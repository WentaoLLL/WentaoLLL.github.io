Abstract

Development of a web-based database system for medical image information

The aim of this master thesis is it to create a web-based database system for medical image information. With the web application the DICOM file can be processed and exchanged fpr medical diagnosis. In advance, an AI application for processing DICOM images was developed and the image data was transferred to the web application via API. Then the image data is transferred to the web application via API. Web application technologies allow recording of medical image data and interaction between doctor and patient in order to exchange diagnosis and process medical images. This web application contains three step positions for administrators, doctors and patients. A DICOM viewer is also used in the web application.

The Solution

Central Questions

These quality requirements are the basis for building a medical image processing system. The patient information is always transmitted or displayed together with image data. In addition, the images belonging to the same patient are combined in one DICOM archive. This means that the DICOM archive is very large because of the numerous and large images

  • Structure of a DICOM viewer for displaying the DICOM archives

The Result

Note - 1.7

With regard to this final project, the requirements were essentially met. A few issues arose during the completion process. However, after careful study and review of relevant article, the issue was resolved. The main points of this work are medical image processing, digital image processing, website creation, data transfer, database construction and image data compression. After some comparisons, Node.js and MongoDB were finally chosen as the basis of the website. In addition, the comfortable Vue.js was chosen as the frontend page framework. Based on this, the medical image processing website including frontend, backend and database was created.

Ohif and Cornerstone.js were compared on the medical image display. Compared to the more complex and mature Ohif framework, Cornerstone.js is more suitable for this project because it is lightweight and easy to implement and its features met the needs of our project. So Ohif was not used because it is more complex and expensive to research. There was a problem with image data transfer because the data volume was too large, which caused the server to run slowly. After measuring the image display quality, it was decided to convert the DCM image to JEPG2000 or PNG format, transfer it and display it. However, the website also retains the ability to accept and display images in DCM format. Once a suitable high-speed server is used, these functions can also be accessed.

After all issues are resolved, the website application runs as desired. All functions basically meet the specified requirements. Some imperfections are improved step by step. In general, the progress of the project can be rated as positive. Finally, this thesis project solidified my knowledge of web development and learned many theories of medical imaging. It provides good support for my future study or work.
